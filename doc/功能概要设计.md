# Fly Music 音乐推荐平台 - 功能概要设计

## 1. 项目概述

### 1.1 项目背景

随着音乐平台的普及，用户面临着海量音乐资源的选择困难。传统的音乐推荐方式（如排行榜、分类浏览）已难以满足用户个性化的需求。本项目旨在开发一款基于用户行为分析的智能音乐推荐平台，通过雷达歌单、热门榜单、听歌报告等功能，为用户提供精准的音乐推荐服务。

### 1.2 项目目标

- 实现基于用户行为的个性化音乐推荐
- 提供多维度热门歌曲榜单
- 生成用户听歌数据分析报告
- 为管理员提供推荐决策支持

### 1.3 技术选型

| 层次 | 技术栈 |
|-----|-------|
| 前端 | Vue 3 + Vite + Element Plus + Pinia |
| 后端 | Spring Boot 3.2.0 + Java 21 |
| 数据库 | MySQL 8.0 |
| ORM | Spring Data JPA |
| 构建工具 | Maven |

## 3. 功能模块详细设计

### 3.1 雷达歌单模块

#### 3.1.1 功能描述

雷达歌单是基于用户历史播放行为，通过分析用户的音乐偏好标签，生成个性化推荐歌单。

#### 3.1.2 核心算法

**用户行为权重**：
| 行为类型 | 权重值 |
|---------|-------|
| 播放 | +1 |
| 播放完成 | +2 |
| 收藏 | +5 |
| 分享 | +3 |
| 跳过 | -3 |
| 不喜欢 | -5 |

**兴趣画像更新**：
- 每日凌晨定时计算用户兴趣画像
- 根据行为权重累加标签分值
- 选取分值最高的标签作为主推荐依据

**歌曲推荐**：
- 匹配用户兴趣标签的歌曲
- 按热度加权排序
- 过滤用户已播放过的歌曲

#### 3.1.3 接口设计

```
GET /api/recommendation/radar
- 功能：获取雷达歌单
- 参数：无
- 返回：推荐歌曲列表

GET /api/recommendation/interest
- 功能：获取用户兴趣画像
- 参数：无
- 返回：兴趣标签及分值

POST /api/recommendation/behavior
- 功能：记录用户行为
- 参数：songId, behaviorType
- 返回：操作结果
```

#### 3.1.4 数据表设计

```sql
-- 用户行为记录表
CREATE TABLE user_behavior (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    song_id BIGINT NOT NULL,
    behavior_type VARCHAR(20) NOT NULL,
    weight INT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用户兴趣画像表
CREATE TABLE user_interest_profile (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL UNIQUE,
    interest_tags JSON,
    last_calculated_at TIMESTAMP
);

-- 雷达歌单表
CREATE TABLE radar_playlist (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    song_id BIGINT NOT NULL,
    score DOUBLE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 热门榜单模块

#### 3.2.1 功能描述

热门榜单展示全站最受欢迎的歌曲，支持多种榜单类型：全局热门榜、分类热门榜、新歌榜、飙升榜。

#### 3.2.2 热度计算算法

**热度衰减公式**：
```
score = plays / (days + 2)^1.8
```

- `plays`: 歌曲播放次数
- `days`: 距离首次上架的天数
- `1.8`: 衰减系数，控制热度下降速度
- `+2`: 防止分母为0

**榜单类型**：
| 榜单类型 | 计算方式 |
|---------|---------|
| 全局热门榜 | 全站所有歌曲按热度排序 |
| 分类热门榜 | 按分类筛选后按热度排序 |
| 新歌榜 | 上架30天内歌曲按热度排序 |
| 飙升榜 | 近7天热度增长最快的歌曲 |

#### 3.2.3 接口设计

```
GET /api/ranking/global
- 功能：获取全局热门榜
- 参数：limit
- 返回：热门歌曲列表

GET /api/ranking/category/{categoryId}
- 功能：获取分类热门榜
- 参数：categoryId, limit
- 返回：分类热门歌曲列表

GET /api/ranking/new
- 功能：获取新歌榜
- 参数：limit
- 返回：新歌列表

GET /api/ranking/soaring
- 功能：获取飙升榜
- 参数：limit
- 返回：飙升歌曲列表
```

#### 3.2.4 数据表设计

```sql
-- 歌曲热度分值表
CREATE TABLE song_hot_score (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    song_id BIGINT NOT NULL UNIQUE,
    hot_score DOUBLE DEFAULT 0,
    play_count BIGINT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 热门榜单表
CREATE TABLE hot_ranking (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    song_id BIGINT NOT NULL,
    ranking_type VARCHAR(20) NOT NULL,
    category_id BIGINT,
    rank_position INT,
    hot_score DOUBLE,
    calculated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.4 听歌报告模块

#### 3.4.1 功能描述

生成用户年度听歌数据报告，包括听歌时长、最喜欢的歌曲、歌手、曲风分布等。

#### 3.4.2 报告内容

| 统计项 | 说明 |
|-------|------|
| 听歌曲数 | 累计播放过的歌曲总数 |
| 听歌时长 | 累计播放总时长（小时） |
| 听歌天数 | 有播放记录的天数 |
| 播放次数 | 累计播放次数 |
| 曲风分布 | 各分类播放占比 |
| Top10歌曲 | 播放次数最多的歌曲 |
| 常听歌手 | 播放最多的歌手 |
| 时间分布 | 各时段播放频率 |

#### 3.4.3 接口设计

```
GET /api/report/listening
- 功能：获取当前年份听歌报告
- 参数：无
- 返回：年度报告数据

GET /api/report/listening/{year}
- 功能：获取指定年份听歌报告
- 参数：year
- 返回：年度报告数据

GET /api/report/summary
- 功能：获取简要统计
- 参数：无
- 返回：基础统计数据
```

### 3.5 推荐决策看板模块（管理端）

#### 3.5.1 功能描述

为管理员提供推荐系统的数据监控和权重配置功能。

#### 3.5.2 功能列表

**数据统计**：
- 全站用户最喜欢的曲风分布
- 全站最受欢迎的歌手 TOP5
- 今日播放次数、听歌时长、活跃用户数
- 总歌曲数、总用户数

**权重配置**：
| 推荐类型 | 权重范围 | 说明 |
|---------|--------|------|
| 雷达歌单权重 | 1-5 | 基于用户行为的推荐 |
| 热门歌单权重 | 1-5 | 基于热度的推荐 |
| 分类推荐权重 | 1-5 | 基于分类的推荐 |
| 相似推荐权重 | 1-5 | 基于相似歌曲的推荐 |

#### 3.5.3 接口设计

```
GET /api/admin/recommendation/dashboard
- 功能：获取推荐看板数据
- 参数：无
- 返回：统计数据

GET /api/admin/recommendation/config
- 功能：获取推荐权重配置
- 参数：无
- 返回：当前权重

PUT /api/admin/recommendation/config
- 功能：更新推荐权重配置
- 参数：radarWeight, hotWeight, categoryWeight, similarWeight
- 返回：操作结果

POST /api/admin/recommendation/recalculate
- 功能：使用新权重重新计算
- 参数：无
- 返回：操作结果
```
