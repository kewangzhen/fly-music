# Fly Music 系统功能设计文档

## 1. 系统概述

### 1.1 项目简介

Fly Music 是一个功能完善的在线音乐平台，采用 Spring Boot + Vue 3 技术栈构建，支持音乐播放、收藏、歌单管理、用户社交、AI 音乐生成、个性化推荐等核心功能。系统采用微服务架构设计，后端提供 RESTful API 接口，前端采用 Element Plus 组件库构建响应式界面。

### 1.2 技术架构

**后端技术栈**：Spring Boot 3.2.0、Spring Security、Spring Data JPA、MySQL 8.0、Redis、Java 21

**前端技术栈**：Vue 3.5.25、Vue Router 4.3.0、Pinia、Element Plus、Axios

**AI 技术**：ACE-Step 1.5 音乐生成模型、MusicGen 本地部署

## 4. 数据流设计

### 4.1 用户数据流

```
用户注册 → UserController.register() → UserService.register() 
         → PasswordEncoder.encode() → UserRepository.save() → 数据库
         
用户登录 → UserController.login() → UserService.login() 
        → UserRepository.findByUsername() → PasswordEncoder.matches() 
        → JwtUtils.generateToken() → 返回 Token
```

### 4.2 音乐播放数据流

```
播放请求 → SongController.playSong() → SongService.incrementPlayCount() 
        → UserBehaviorRepository.save() (记录播放行为)
        → UserInterestProfileRepository.update() (更新用户画像)
        → SongHotScoreRepository.update() (更新热度分数)
```

### 4.3 推荐数据流

```
推荐请求 → RecommendationController.guessYouLike() 
         → RecommendationService.getGuessYouLike() 
         → UserBehaviorRepository.findByUserId() (获取用户行为)
         → UserInterestProfileRepository.findByUserId() (获取用户画像)
         → 计算推荐分数 → SongRepository.findByIds() (获取推荐歌曲)
         → 返回推荐结果
```

## 6. 关键业务流程

### 6.1 用户完整听歌流程

```
1. 用户登录 → 获取 JWT Token
2. 浏览歌曲 → 搜索/推荐/分类
3. 播放歌曲 → 增加播放次数 → 记录播放历史 → 记录用户行为 → 更新用户画像
4. 收藏歌曲 → 添加到收藏 → 更新收藏计数
5. 分享歌曲 → 增加分享次数 → 记录分享行为
6. 生成推荐 → 基于用户画像 → 返回推荐歌曲
```

### 6.2 AI 音乐生成流程

```
1. 用户输入提示词 → AI Music Controller
2. 保存生成记录（状态：生成中）
3. 调用 ACE-Step / MusicGen API
4. AI 服务生成音乐
5. 获取生成结果
6. 更新生成记录（状态：成功/失败）
7. 返回生成结果给用户
```
