# 🎤 热门榜单功能设计文档

## 一、需求概述

热门榜单是一个基于全局用户行为的歌曲热度排行榜，通过对所有用户的行为数据进行加权计算，结合热度衰减公式，生成客观公正的热门歌曲榜单。

### 核心特性
- ✅ 每日定时计算更新
- ✅ 基于全部用户行为
- ✅ 多维度指标加权
- ✅ 热度时间衰减

---

## 二、算法设计

### 2.1 行为权重定义

| 行为类型 | 权重分值 | 说明 |
|---------|---------|------|
| 播放一次 | +1 | 用户播放歌曲 |
| 收藏一次 | +5 | 用户收藏歌曲 |
| 分享一次 | +8 | 用户分享歌曲 |
| 下载一次 | +3 | 用户下载歌曲 |
| 评论一次 | +4 | 用户评论歌曲 |

### 2.2 热度衰减公式

为了体现"时间越近热度越高"的原则，引入热度衰减：

```
热度得分 = 加权总分 / (发布时间(天) + 2)^G
```

其中：
- **发布时间(天)**: 歌曲发布距今天数
- **G (Gravity)**: 衰减因子，默认值为 1.8

### 2.3 计算示例

假设有一首歌曲A，发布于3天前：
- 播放次数: 1000 → 得分: 1000 × 1 = 1000
- 收藏次数: 100 → 得分: 100 × 5 = 500
- 分享次数: 50 → 得分: 50 × 8 = 400
- 下载次数: 30 → 得分: 30 × 3 = 90
- 评论次数: 20 → 得分: 20 × 4 = 80

**加权总分** = 1000 + 500 + 400 + 90 + 80 = 2070

**热度衰减** = 2070 / (3 + 2)^1.8 = 2070 / 13.38 ≈ **154.7**

## 三、榜单类型设计

### 3.1 榜单分类

| 榜单名称 | 说明 | 更新时间 |
|---------|------|---------|
| 全局热门榜 | 所有歌曲的热门排行 | 每日 |

## 六、算法实现设计

### 6.1 热度计算方法

```java
public void calculateHotScores() {
    List<Song> songs = songRepository.findAll();
    
    for (Song song : songs) {
        // 1. 计算各维度加权得分
        int playScore = song.getPlayCount() * WEIGHT_PLAY;
        int favoriteScore = getFavoriteCount(song.getId()) * WEIGHT_FAVORITE;
        int shareScore = getShareCount(song.getId()) * WEIGHT_SHARE;
        int downloadScore = song.getDownloadCount() * WEIGHT_DOWNLOAD;
        int commentScore = getCommentCount(song.getId()) * WEIGHT_COMMENT;
        
        long totalWeightedScore = playScore + favoriteScore + shareScore 
                               + downloadScore + commentScore;
        
        // 2. 计算热度衰减
        double hotScore = calculateHotScore(totalWeightedScore, song.getCreatedAt());
        
        // 3. 保存热度统计
        saveHotScore(song.getId(), playScore, favoriteScore, shareScore, 
                    downloadScore, commentScore, totalWeightedScore, hotScore);
    }
}

private double calculateHotScore(long weightedScore, Date createdAt) {
    long daysSinceCreation = getDaysBetween(createdAt, new Date());
    double gravity = 1.8;
    double decayFactor = Math.pow(daysSinceCreation + 2, gravity);
    return weightedScore / decayFactor;
}
```

### 6.2 榜单生成方法

```java
public void generateRankings() {
    // 1. 生成全局热门榜
    generateGlobalRanking();
    
    // 2. 生成分类热门榜
    generateCategoryRankings();
    
    // 3. 生成新歌榜
    generateNewSongRanking();
}

private void generateGlobalRanking() {
    List<SongHotScore> hotScores = songHotScoreRepository
        .findAllByOrderByHotScoreDesc(PageRequest.of(0, 100));
    
    for (int i = 0; i < hotScores.size(); i++) {
        saveRanking("global", null, hotScores.get(i).getSongId(), 
                   i + 1, hotScores.get(i).getHotScore());
    }
}

private void generateCategoryRankings() {
    List<Category> categories = categoryRepository.findAll();
    
    for (Category category : categories) {
        List<SongHotScore> hotScores = songHotScoreRepository
            .findByCategoryOrderByHotScoreDesc(category.getId(), 
                                              PageRequest.of(0, 100));
        
        for (int i = 0; i < hotScores.size(); i++) {
            saveRanking("category", category.getId(), hotScores.get(i).getSongId(),
                       i + 1, hotScores.get(i).getHotScore());
        }
    }
}
```

## 七、定时任务设计

### 7.1 任务调度

| 任务 | 执行时间 | 说明 |
|------|---------|------|
| 热度得分计算 | 每日 2:30 | 计算所有歌曲热度 |
| 全局热门榜生成 | 每日 3:00 | 生成Top100 |
| 分类热门榜生成 | 每日 3:30 | 按分类生成 |
| 新歌榜生成 | 每日 4:00 | 最近7天新歌 |
| 飙升榜更新 | 每小时 | 24小时内热度上升 |

### 7.2 代码示例

```java
@Scheduled(cron = "0 30 2 * * ?")
public void dailyCalculateHotScores() {
    log.info("开始计算歌曲热度得分...");
    calculateHotScores();
    log.info("歌曲热度得分计算完成");
}

@Scheduled(cron = "0 0 3 * * ?")
public void dailyGenerateGlobalRanking() {
    log.info("开始生成全局热门榜...");
    generateGlobalRanking();
    log.info("全局热门榜生成完成");
}

@Scheduled(cron = "0 30 3 * * ?")
public void dailyGenerateCategoryRankings() {
    log.info("开始生成分类热门榜...");
    generateCategoryRankings();
    log.info("分类热门榜生成完成");
}

@Scheduled(cron = "0 0 4 * * ?")
public void dailyGenerateNewSongRanking() {
    log.info("开始生成新歌榜...");
    generateNewSongRanking();
    log.info("新歌榜生成完成");
}

@Scheduled(cron = "0 0 * * * ?")
public void hourlyUpdateSoaringRanking() {
    log.info("开始更新飙升榜...");
    generateSoaringRanking();
    log.info("飙升榜更新完成");
}
```

## 十、后续优化建议

### 10.1 算法优化

1. **引入时间因子**: 当前使用简单的时间衰减，可考虑小时级衰减
2. **用户质量加权**: 活跃用户的行为权重更高
3. **防刷机制**: 识别异常数据，排除机器刷量

### 10.2 功能扩展

1. **多维度排序**: 支持按播放量、收藏量、分享量单独排序
2. **个性化榜单**: 结合用户兴趣的定制榜单
3. **实时榜单**: WebSocket推送的实时热度变化
