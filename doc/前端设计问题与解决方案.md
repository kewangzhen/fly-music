# Fly-Music 前端设计问题与解决方案

## 一、技术基础问题

### 问题1：Vue 3 的 Composition API 不熟悉
**解决方案：** 建议先系统学习 Vue 3 官方文档中的 Composition API 部分，掌握 `ref`、`reactive`、`computed`、`watch` 等核心 API 的使用。可以先用 Options API 编写简单组件，再逐步迁移到 Composition API。

### 问题2：组件间通信混乱
**解决方案：** 
- 父子通信：使用 `props` + `emit`
- 跨层级通信：使用 `provide` / `inject`
- 复杂状态：使用 Pinia 状态管理

---

## 二、登录与权限问题

### 问题3：刷新页面后登录状态丢失
**解决方案：** 在 `localStorage` 中存储用户信息，页面加载时从 localStorage 读取并恢复状态。应用入口处添加状态初始化逻辑。

### 问题4：Token 过期如何处理
**解决方案：** 使用 Axios 响应拦截器，捕获 401 错误。方案一：跳转登录页；方案二：调用刷新 Token 接口，成功后重新发起原请求。

### 问题5：未登录用户访问需要权限的页面
**解决方案：** 使用 Vue Router 的路由守卫 `beforeEach`，在全局守卫中检查用户登录状态和 Token，未登录则重定向到登录页。

---

## 三、API 请求问题

### 问题6：搜索时请求过于频繁
**解决方案：** 使用防抖（Debounce）函数，延迟 300-500ms 发送请求。可以用 Lodash 的 `debounce` 或手写防抖函数。

### 问题7：API 请求失败的错误提示不友好
**解决方案：** 在 Axios 响应拦截器中统一处理错误，弹出 Element Plus 的 `ElMessage` 提示。区分网络错误、业务错误、服务器错误，给出不同的提示文案。

### 问题8：大数据量分页加载慢
**解决方案：** 
- 后端实现分页，前端使用分页组件
- 考虑使用无限滚动（Infinite Scroll）代替传统分页
- 对热门数据做本地缓存

---

## 四、音乐播放问题

### 问题9：页面刷新后播放进度丢失
**解决方案：** 使用 Pinia 或 Vuex 全局状态管理播放器状态，包括当前歌曲、播放进度、播放状态。刷新后从最近播放历史恢复。

### 问题10：多个页面需要控制播放
**解决方案：** 创建全局播放器组件，固定在页面底部。播放器组件监听全局状态变化，通过 Pinia 共享播放状态，实现跨页面控制。

### 问题11：播放列表如何管理
**解决方案：** 在 Pinia 中创建播放列表状态管理，包含列表数据、当前索引、播放模式（顺序/随机/循环）。提供添加、删除、切换歌曲的方法。

---

## 五、性能优化问题

### 问题12：首屏加载太慢
**解决方案：**
- 路由懒加载：`const Home = () => import('./views/Home.vue')`
- Element Plus 按需引入
- 图片使用 CDN 或懒加载
- 开启 Gzip 压缩

### 问题13：长列表滚动卡顿
**解决方案：** 使用虚拟滚动（Virtual Scroller）技术，只渲染可视区域内的元素。对于歌曲列表，可以限制一次性渲染的数量。

### 问题14：重复请求相同数据
**解决方案：** 对热门数据（如榜单、推荐歌单）做内存缓存，设置过期时间。避免用户切换页面时重复请求相同数据。

---

## 六、数据展示问题

### 问题15：后端返回的时间格式不方便展示
**解决方案：** 使用 Day.js 或 Moment.js 统一处理。在 axios 响应拦截器中全局转换，或在组件中 computed 属性中格式化。

### 问题16：复杂的嵌套数据结构难以解析
**解决方案：** 在组件的 `setup` 中编写数据转换函数，将后端返回的嵌套结构转换为平面结构，便于模板渲染。

### 问题17：图片加载失败显示空白
**解决方案：** 给 `<img>` 标签添加 `@error` 事件，加载失败时使用默认占位图。同时在 CSS 中设置默认背景色。

---

## 七、表单与验证问题

### 问题18：表单验证规则复杂
**解决方案：** 使用 Element Plus 的 Form 验证，结合自定义校验规则。对于复杂表单，拆分为多个子表单组件，降低复杂度。

### 问题19：文件上传进度显示
**解决方案：** 使用 Axios 的 `onUploadProgress` 事件监听上传进度，更新进度条组件。AI 音乐生成等耗时操作可使用轮询或 WebSocket 接收进度。

---

## 八、社交功能问题

### 问题20：点赞状态同步问题
**解决方案：** 使用乐观更新（先更新 UI，再发请求），请求失败则回滚状态。同时防止快速连续点击，使用防抖处理。

### 问题21：关注列表操作后页面不更新
**解决方案：** 操作成功后立即更新本地状态（Pinia 中的关注列表），或者重新调用接口刷新数据，保持前后端状态一致。

---

## 九、项目架构问题

### 问题22：代码组织混乱
**解决方案：** 按功能模块划分目录，如 `views/home/`、`views/user/`、`components/common/`。API 接口按模块拆分到不同文件。

### 问题23：组件命名不规范
**解决方案：** 制定命名规范，如：页面组件用 `Page.vue`，列表组件用 `List.vue`，卡片用 `Card.vue`。保持团队命名风格一致。

---

## 十、调试与兼容问题

### 问题24：控制台大量警告
**解决方案：** 仔细阅读 Element Plus 升级提示，逐步修复 deprecated 的 API 用法。可以在控制台过滤警告，逐步排查。

### 问题25：移动端适配困难
**解决方案：** 使用媒体查询（@media）处理响应式布局。Element Plus 提供 `hidden-xs`、`hidden-sm` 等类名辅助隐藏元素。

---

## 十一、部署问题

### 问题26：构建后的文件部署到服务器404
**解决方案：** Vue Router 使用 history 模式时，需要服务器配置伪静态支持。Nginx 中添加 `try_files $uri $uri/ /index.html` 配置。

### 问题27：CDN 资源加载失败
**解决方案：** 保留本地 fallback 方案，CDN 失败时加载本地资源。可以在 index.html 中预先加载关键资源。
