# 个性化音乐推荐系统数据库设计文档

## 1. 数据库概览

本数据库设计基于MySQL 8.0，用于支持个性化音乐推荐系统的所有功能模块，包括用户管理、音乐管理、播放管理、歌单管理、个性化推荐、AI音乐实验室和社交互动等核心功能。

## 2. 核心表结构设计

### 2.1 用户表 (users)

| 字段名             | 数据类型           | 约束                                                      | 描述                       |
| :-------------- | :------------- | :------------------------------------------------------ | :----------------------- |
| `id`            | `BIGINT`       | `PRIMARY KEY, AUTO_INCREMENT`                           | 用户ID                     |
| `username`      | `VARCHAR(50)`  | `UNIQUE, NOT NULL`                                      | 用户名                      |
| `password`      | `VARCHAR(255)` | `NOT NULL`                                              | 密码（加密存储）                 |
| `email`         | `VARCHAR(100)` | `UNIQUE, NOT NULL`                                      | 邮箱                       |
| `phone`         | `VARCHAR(20)`  | `UNIQUE, NOT NULL`                                      | 手机号                      |
| `avatar`        | `VARCHAR(255)` | <br />                                                  | 头像URL                    |
| `gender`        | `TINYINT`      | <br />                                                  | 性别（0:未知, 1:男, 2:女）       |

| `role`          | `TINYINT`      | `DEFAULT 0`                                             | 角色（0:普通用户, 1:VIP, 2:管理员） |
| `status`        | `TINYINT`      | `DEFAULT 1`                                             | 状态（0:禁用, 1:正常）           |
| `vip_expire_at` | `TIMESTAMP`    | <br />                                                  | VIP过期时间                  |
| `created_at`    | `TIMESTAMP`    | `DEFAULT CURRENT_TIMESTAMP`                             | 创建时间                     |

## 3. 索引设计

### 3.1 主键索引

* 所有表的`id`字段均为主键，自动创建主键索引

### 3.2 唯一索引

* `users.username` - 确保用户名唯一

* `users.email` - 确保邮箱唯一

* `users.phone` - 确保手机号唯一

* `user_follows(follower_id, followed_id)` - 确保用户只能关注另一个用户一次

### 3.3 普通索引

* `songs.album_id` - 加速按专辑查询歌曲

* `songs.category_id` - 加速按分类查询歌曲

* `playlists.user_id` - 加速查询用户创建的歌单

* `play_history.user_id` - 加速查询用户播放历史

* `play_history.song_id` - 加速查询歌曲的播放历史

* `user_favorites.user_id` - 加速查询用户收藏

* `user_favorites.target_type` - 加速按类型查询收藏

* `comments.target_type` - 加速按类型查询评论

* `comments.target_id` - 加速查询特定对象的评论

* `posts.user_id` - 加速查询用户发布的动态

* `posts.song_id` - 加速查询关联歌曲的动态

* `ai_music_generations.user_id` - 加速查询用户的AI音乐生成记录

### 3.4 复合索引

* `play_history(user_id, played_at)` - 加速查询用户最近播放历史

* `user_favorites(user_id, target_type, target_id)` - 确保用户对同一对象只收藏一次

* `playlist_songs(playlist_id, order_index)` - 加速按顺序查询歌单歌曲

* `comments(target_type, target_id, created_at)` - 加速查询特定对象的最新评论
