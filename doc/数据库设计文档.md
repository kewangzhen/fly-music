# Fly Music 数据库设计文档

## 1. 数据库概述

本数据库设计文档描述了 Fly Music 音乐平台的数据库结构，包括用户管理、音乐管理、歌单管理、用户行为分析和推荐系统等核心功能。数据库采用 MySQL 8.0，使用 Spring Data JPA 作为 ORM 框架。

## 4. 索引设计

### 4.1 主键索引
- 所有表的 `id` 字段均为主键，自动创建主键索引

### 4.2 唯一索引
- `users.username` - 确保用户名唯一
- `users.email` - 确保邮箱唯一
- `users.phone` - 确保手机号唯一
- `system_configs.config_key` - 确保配置键唯一
- `user_interest_profile.user_id` - 确保每个用户只有一个兴趣画像
- `song_hot_score.song_id` - 确保每个歌曲只有一个热度分数

### 4.3 普通索引
- `songs.album_id` - 加速按专辑查询歌曲
- `songs.category_id` - 加速按分类查询歌曲
- `songs.user_id` - 加速查询用户上传的歌曲
- `albums.artist_id` - 加速查询艺术家的专辑
- `playlists.user_id` - 加速查询用户的歌单
- `user_behavior.user_id` - 加速查询用户行为
- `user_behavior.song_id` - 加速查询歌曲的用户行为
- `play_histories.user_id` - 加速查询用户播放历史
- `play_histories.song_id` - 加速查询歌曲的播放历史
- `user_favorites.user_id` - 加速查询用户收藏
- `user_favorites.song_id` - 加速查询歌曲的收藏情况
- `hot_ranking.type` - 加速按类型查询排行榜
- `hot_ranking.song_id` - 加速查询歌曲的排行情况
- `ai_music_generation.user_id` - 加速查询用户的AI音乐生成记录
- `posts.user_id` - 加速查询用户的帖子
- `post_comments.post_id` - 加速查询帖子的评论

- `comments.song_id` - 加速查询歌曲的评论
- `comments.user_id` - 加速查询用户的评论

### 4.4 复合索引
- `user_behavior (user_id, song_id)` - 加速查询用户对特定歌曲的行为

- `hot_ranking (type, rank)` - 加速查询特定类型的排行榜

## 6. 性能优化建议

### 6.1 数据库层面
1. **使用合适的存储引擎**：InnoDB 存储引擎，支持事务和外键
2. **分区表**：对于数据量较大的表（如 user_behavior、play_histories），考虑使用分区表
3. **分表**：对于热门排行榜等数据，可以考虑按时间分表
4. **缓存**：使用 Redis 缓存热点数据，如排行榜、热门歌曲等
5. **读写分离**：在高并发场景下，考虑使用读写分离架构

### 6.2 SQL 优化
1. **索引优化**：合理创建和使用索引，避免全表扫描
2. **查询优化**：使用 `EXPLAIN` 分析查询计划，优化复杂查询
3. **批量操作**：对于批量插入、更新操作，使用批量处理减少数据库交互
4. **分页查询**：使用 `LIMIT` 进行分页，避免一次性查询大量数据
5. **连接查询**：合理使用连接查询，避免过多的表连接

### 6.3 应用层面
1. **延迟加载**：使用 JPA 的延迟加载机制，避免加载不必要的数据
2. **批量处理**：对于大量数据的操作，使用批量处理
3. **缓存策略**：实现多级缓存策略，减少数据库访问
4. **异步处理**：对于非实时性操作，使用异步处理
5. **数据清理**：定期清理过期数据，保持数据库健康

## 7. 数据安全

### 7.1 密码安全
- 密码使用 BCrypt 等安全算法加密存储
- 禁止明文存储密码
- 实现密码强度验证

### 7.2 数据访问控制
- 实现基于角色的访问控制（RBAC）
- 限制敏感数据的访问权限
- 对敏感操作进行审计日志记录

### 7.3 SQL 注入防护
- 使用参数化查询，避免 SQL 注入
- 使用 ORM 框架的安全特性
- 对输入数据进行严格验证

### 7.4 数据备份
- 定期进行数据库备份
- 实现增量备份策略
- 测试备份恢复流程
