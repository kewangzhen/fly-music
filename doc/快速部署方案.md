# Fly Music 快速部署方案

本文档提供多种快速部署方案，您可以根据实际情况选择最适合的方案。

---

## 方案一：Docker 一键部署（推荐）

### 1.1 方案概述

使用 Docker Compose 一键部署所有服务，包括：
- MySQL 8.0 数据库
- Redis 缓存
- Fly Music 后端服务
- Fly Music 前端服务

### 1.2 前置要求

- 安装 Docker Desktop：https://www.docker.com/products/docker-desktop
- 至少 4GB 可用内存
- 至少 20GB 可用磁盘空间

### 1.3 部署步骤

#### 步骤 1：创建部署目录

```powershell
# 创建部署目录
mkdir -p fly-music-deploy
cd fly-music-deploy
```

#### 步骤 2：下载部署文件

将以下 `docker-compose.yml` 文件保存到部署目录：

```yaml
version: '3.8'

services:
  # MySQL 数据库
  mysql:
    image: mysql:8.0
    container_name: fly-music-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root123456
      MYSQL_DATABASE: fly_music
      MYSQL_USER: flymusic
      MYSQL_PASSWORD: flymusic123
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: fly-music-redis
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Fly Music 后端
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: fly-music-backend
    restart: always
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/fly_music?useUnicode=true&characterEncoding=utf-8&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai
      SPRING_DATASOURCE_USERNAME: flymusic
      SPRING_DATASOURCE_PASSWORD: flymusic123
      SPRING_DATA_REDIS_HOST: redis
      SPRING_DATA_REDIS_PORT: 6379
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    volumes:
      - ./songs:/app/songs
      - ./covers:/app/covers

  # Fly Music 前端
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: fly-music-frontend
    restart: always
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      - backend

volumes:
  mysql_data:
  redis_data:
```

#### 步骤 3：创建后端 Dockerfile

在 `backend` 目录创建 `Dockerfile`：

```dockerfile
FROM maven:3.9-eclipse-temurin-21 AS builder

WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

FROM eclipse-temurin:21-jre
WORKDIR /app
COPY --from=builder /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 步骤 4：创建前端 Dockerfile

在 `frontend` 目录创建 `Dockerfile`：

```dockerfile
FROM node:20-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf
EXPOSE 80 443
CMD ["nginx", "-g", "daemon off;"]
```

#### 步骤 5：启动服务

```powershell
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

#### 步骤 6：访问应用

- 前端：http://localhost
- 后端 API：http://localhost:8080
- MySQL：localhost:3306
- Redis：localhost:6379

## 方案二：手动部署（适用于无 Docker 环境）

### 2.1 前置要求

| 软件 | 版本要求 | 下载地址 |
|------|---------|---------|
| JDK | 21+ | https://adoptium.net/ |
| Maven | 3.9+ | https://maven.apache.org/ |
| Node.js | 20+ | https://nodejs.org/ |
| MySQL | 8.0+ | https://www.mysql.com/ |
| Redis | 7.0+ | https://redis.io/ |

### 2.2 一键安装脚本

创建 `install-env.ps1`（Windows PowerShell）：

```powershell
# Fly Music 环境安装脚本 - Windows

Write-Host "=== Fly Music 环境安装脚本 ===" -ForegroundColor Green

# 1. 安装 Chocolatey（如果未安装）
if (!(Get-Command choco -ErrorAction SilentlyContinue)) {
    Write-Host "安装 Chocolatey..." -ForegroundColor Yellow
    Set-ExecutionPolicy Bypass -Scope Process -Force
    [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072
    Invoke-Expression ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
}

# 2. 安装 JDK 21
Write-Host "安装 JDK 21..." -ForegroundColor Yellow
choco install openjdk21 -y

# 3. 安装 Maven
Write-Host "安装 Maven..." -ForegroundColor Yellow
choco install maven -y

# 4. 安装 Node.js
Write-Host "安装 Node.js 20..." -ForegroundColor Yellow
choco install nodejs20 -y

# 5. 安装 MySQL
Write-Host "安装 MySQL 8.0..." -ForegroundColor Yellow
choco install mysql -y

# 6. 安装 Redis
Write-Host "安装 Redis..." -ForegroundColor Yellow
choco install redis-64 -y

Write-Host "=== 环境安装完成 ===" -ForegroundColor Green
Write-Host "请重启终端后执行项目启动脚本" -ForegroundColor Cyan
```

### 2.3 项目启动脚本

创建 `start-flymusic.ps1`：

```powershell
# Fly Music 项目启动脚本 - Windows

Write-Host "=== 启动 Fly Music ===" -ForegroundColor Green

# 配置参数
$MYSQL_ROOT_PASSWORD = "root123456"
$MYSQL_DATABASE = "fly_music"
$MYSQL_USER = "flymusic"
$MYSQL_PASSWORD = "flymusic123"

# 1. 启动 MySQL
Write-Host "启动 MySQL..." -ForegroundColor Yellow
Start-Service MySQL
mysql -u root -p$MYSQL_ROOT_PASSWORD -e "CREATE DATABASE IF NOT EXISTS $MYSQL_DATABASE;"
mysql -u root -p$MYSQL_ROOT_PASSWORD -e "CREATE USER IF NOT EXISTS '$MYSQL_USER'@'localhost' IDENTIFIED BY '$MYSQL_PASSWORD';"
mysql -u root -p$MYSQL_ROOT_PASSWORD -e "GRANT ALL PRIVILEGES ON $MYSQL_DATABASE.* TO '$MYSQL_USER'@'localhost';"
mysql -u root -p$MYSQL_ROOT_PASSWORD -e "FLUSH PRIVILEGES;"

# 2. 启动 Redis
Write-Host "启动 Redis..." -ForegroundColor Yellow
Start-Service Redis

# 3. 导入数据库
Write-Host "导入数据库..." -ForegroundColor Yellow
mysql -u $MYSQL_USER -p$MYSQL_PASSWORD $MYSQL_DATABASE < fly_music_data.sql

# 4. 修改配置文件
Write-Host "修改配置文件..." -ForegroundColor Yellow
$env:SPRING_DATASOURCE_URL = "jdbc:mysql://localhost:3306/$MYSQL_DATABASE?useUnicode=true&characterEncoding=utf-8&useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai"
$env:SPRING_DATASOURCE_USERNAME = $MYSQL_USER
$env:SPRING_DATASOURCE_PASSWORD = $MYSQL_PASSWORD

# 5. 启动后端
Write-Host "启动后端服务..." -ForegroundColor Yellow
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd backend; mvn spring-boot:run"

# 6. 启动前端
Write-Host "启动前端服务..." -ForegroundColor Yellow
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd frontend; npm install; npm run dev"

Write-Host "=== Fly Music 启动完成 ===" -ForegroundColor Green
Write-Host "前端地址: http://localhost:5173" -ForegroundColor Cyan
Write-Host "后端地址: http://localhost:8080" -ForegroundColor Cyan
```

## 方案三：便携版部署（推荐用于快速测试）

### 3.1 方案概述

将所有依赖和环境打包成一个便携版，直接解压即可运行。

### 3.2 目录结构

```
fly-music-portable/
├── jdk/                    # JDK 21 便携版
├── maven/                   # Maven 3.9 便携版
├── node/                   # Node.js 20 便携版
├── mysql/                  # MySQL 8.0 便携版
├── redis/                  # Redis 7.0 便携版
├── songs/                  # 歌曲存储目录
├── covers/                 # 封面存储目录
├── backend/                # 后端项目
├── frontend/               # 前端项目
├── data/                   # 数据库数据
├── start.bat               # 一键启动脚本
└── README.md              # 使用说明
```

### 3.3 便携版制作

```powershell
# 制作便携版脚本
Write-Host "=== 制作 Fly Music 便携版 ===" -ForegroundColor Green

# 1. 下载便携版软件
# JDK: https://adoptium.net/temurin/releases/
# Maven: https://maven.apache.org/download.cgi
# Node.js: https://nodejs.org/dist/
# MySQL: https://downloads.mysql.com/archives/community/
# Redis: https://github.com/tporadowski/redis/releases

# 2. 配置环境变量
$env:JDK_HOME = ".\jdk"
$env:MAVEN_HOME = ".\maven"
$env:NODE_HOME = ".\node"
$env:PATH = "$env:JDK_HOME\bin;$env:MAVEN_HOME\bin;$env:NODE_HOME;$env:PATH"

# 3. 打包
Compress-Archive -Path * -DestinationPath fly-music-portable.zip

Write-Host "=== 便携版制作完成 ===" -ForegroundColor Green
```

## 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| Docker 一键部署 | 简单、快速、环境隔离 | 需要 Docker 基础 | 生产环境、快速部署 |
| 手动部署 | 可自定义配置 | 需要手动安装环境 | 开发环境、无 Docker |
| 便携版 | 无需安装、即插即用 | 体积较大 | 演示、快速测试 |

## 常见问题

### Q1: Docker 启动失败

```powershell
# 检查 Docker 状态
docker ps

# 重启 Docker
docker-compose restart
```

### Q2: 数据库连接失败

```powershell
# 检查 MySQL 状态
docker-compose logs mysql

# 重新初始化数据库
docker-compose down -v
docker-compose up -d
```

### Q3: 前端无法访问后端

```powershell
# 修改前端配置
# frontend/.env.development
VITE_API_BASE_URL=http://localhost:8080
```
