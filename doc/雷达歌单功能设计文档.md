# 🎵 雷达歌单功能设计文档

## 一、需求概述

雷达歌单是一个基于用户行为个性化推荐的功能，通过分析用户的历史行为（播放、收藏、完整收听等），计算用户对不同音乐风格/标签的偏好度，从而生成个性化的推荐歌单。

### 核心特性
- ✅ 每日定时计算更新
- ✅ 基于用户历史行为
- ✅ 多维度权重评分
- ✅ 实时兴趣画像

---

## 二、算法设计

### 2.1 行为权重定义

| 行为类型 | 权重分值 | 说明 |
|---------|---------|------|
| 播放歌曲 | +1 | 用户点击播放歌曲 |
| 完整收听 | +2 | 播放完成度 > 90% |
| 收藏歌曲 | +5 | 用户主动收藏 |
| 分享歌曲 | +3 | 用户分享给他人 |
| 切歌/跳过 | -3 | 未完整收听就切换 |
| 不喜欢 | -5 | 用户标记不喜欢 |

### 2.2 标签分数计算流程

```
用户行为 → 歌曲标签提取 → 权重累加 → 兴趣画像
```

**Step 1: 行为记录**
- 用户每执行一次行为，记录到 `user_behavior` 表

**Step 2: 分数计算**
```
歌曲得分 = Σ(行为权重 × 时间衰减因子)
```

**Step 3: 标签聚合**
```
标签得分 = Σ(歌曲得分)  // 同一标签下的所有歌曲分数累加
```

**Step 4: 生成雷达**
```
用户兴趣画像 = {
  "摇滚": 45,
  "民谣": 22,
  "流行": 4,
  "电子": -2
}
```

### 2.3 时间衰减因子

为了体现用户最近的音乐偏好，采用时间衰减：
```
权重 = 基础权重 × (1 / (1 + days × 0.1))
```
- 7天前的行为：权重 × 0.59
- 30天前的行为：权重 × 0.25

## 三、数据库设计

### 3.1 新增表：用户行为记录表

```sql
CREATE TABLE user_behavior (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    song_id BIGINT NOT NULL,
    behavior_type TINYINT NOT NULL COMMENT '1-播放 2-完整收听 3-收藏 4-分享 5-切歌 6-不喜欢',
    weight INT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_behavior (user_id, created_at)
```

### 3.：用户兴趣画像2 新增表

```sql
CREATE TABLE user_interest_profile (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL UNIQUE,
    interest_data JSON NOT NULL COMMENT '{"摇滚": 45, "民谣": 22, ...}',
    last_updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 3.3 新增表：雷达歌单

```sql
CREATE TABLE radar_playlist (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    song_ids JSON NOT NULL COMMENT '推荐的歌曲ID列表',
    interest_summary JSON NOT NULL COMMENT '兴趣摘要，如{"top_tag": "摇滚", "scores": {...}}',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP NOT NULL COMMENT '过期时间（次日）',
    INDEX idx_radar_user (user_id),
    INDEX idx_radar_expires (expires_at)
);
```

## 五、服务层设计

### 5.1 核心方法

#### 方法1: 记录用户行为
```java
public void recordBehavior(Long userId, Long songId, Integer behaviorType) {
    // 1. 获取权重
    int weight = BehaviorType.fromCode(behaviorType).getWeight();
    
    // 2. 保存行为记录
    UserBehavior behavior = new UserBehavior();
    behavior.setUserId(userId);
    behavior.setSongId(songId);
    behavior.setBehaviorType(behaviorType);
    behavior.setWeight(weight);
    behavior.setCreatedAt(new Date());
    
    userBehaviorRepository.save(behavior);
}
```

#### 方法2: 计算用户兴趣画像
```java
public void calculateUserInterestProfile(Long userId) {
    // 1. 获取用户最近30天的行为记录
    Calendar cal = Calendar.getInstance();
    cal.add(Calendar.DAY_OF_MONTH, -30);
    Date startDate = cal.getTime();
    
    List<Object[]> songScores = userBehaviorRepository.calculateSongScores(userId, startDate);
    
    // 2. 按歌曲分组，累加权重
    Map<String, Double> interestProfile = new HashMap<>();
    for (Object[] row : songScores) {
        Long songId = (Long) row[0];
        Integer totalScore = ((Number) row[1]).intValue();
        
        Song song = songRepository.findById(songId);
        if (song.getCategory() != null) {
            String tag = song.getCategory().getName();
            interestProfile.merge(tag, totalScore * 1.0, Double::sum);
        }
    }
    
    // 3. 保存兴趣画像
    saveUserInterestProfile(userId, interestProfile);
}
```

#### 方法3: 生成雷达歌单
```java
public void generateRadarPlaylist(Long userId) {
    // 1. 获取用户兴趣画像
    Map<String, Double> profile = getUserInterests(userId);
    
    // 2. 找出Top 3兴趣标签
    List<Map.Entry<String, Double>> sorted = profile.entrySet().stream()
        .sorted(Map.Entry.comparingByValue().reversed())
        .limit(3)
        .collect(Collectors.toList());
    
    // 3. 根据Top标签查询候选歌曲
    List<Song> candidates = new ArrayList<>();
    for (Map.Entry<String, Double> entry : sorted) {
        String tag = entry.getKey();
        List<Song> songs = songRepository.findByCategoryNameOrderByPlayCountDesc(
            tag, PageRequest.of(0, 20));
        candidates.addAll(songs);
    }
    
    // 4. 去重并选取前20首
    List<Song> finalSongs = candidates.stream()
        .distinct()
        .limit(20)
        .collect(Collectors.toList());
    
    // 5. 保存雷达歌单
    RadarPlaylist radar = new RadarPlaylist();
    radar.setUserId(userId);
    radar.setSongIds(toJson(finalSongs.stream().map(Song::getId).collect(Collectors.toList())));
    radar.setInterestSummary(toJson(profile));
    radar.setExpiresAt(DateUtil.addDays(new Date(), 1));
    
    radarPlaylistRepository.save(radar);
}
```

## 六、定时任务设计

### 6.1 每日凌晨执行

```java
@Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨2点执行
public void dailyRadarGeneration() {
    log.info("开始生成每日雷达歌单...");
    
    // 1. 查询最近30天活跃用户
    Calendar cal = Calendar.getInstance();
    cal.add(Calendar.DAY_OF_MONTH, -30);
    Date thirtyDaysAgo = cal.getTime();
    List<User> activeUsers = userRepository.findActiveUsers(thirtyDaysAgo);
    
    // 2. 为每个用户生成雷达歌单
    for (User user : activeUsers) {
        try {
            generateRadarPlaylist(user.getId());
        } catch (Exception e) {
            log.error("生成用户 {} 雷达歌单失败", user.getId(), e);
        }
    }
    
    // 3. 清理过期雷达歌单
    radarPlaylistRepository.deleteExpired(new Date());
    
    log.info("雷达歌单生成完成");
}
```

## 十、后续优化建议

1. **时间衰减优化**: 目前使用简单的30天窗口，可考虑更精细的指数衰减
2. **协同过滤**: 可结合基于用户的协同过滤算法
3. **内容过滤**: 基于歌曲特征的推荐
4. **实时计算**: 目前是每日批量，可考虑实时更新画像
5. **热门歌曲加权**: 在推荐中加入全局热门歌曲
